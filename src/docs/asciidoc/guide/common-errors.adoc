[#common-error-codes]
=== 공통 에러 코드

에러 응답은 기본적으로 아래 형태를 따릅니다.

[source,json]
----
{
  "code": "AUTH_UNAUTHORIZED",
  "message": "인증이 필요합니다.",
  "timestamp": "2026-02-20T12:34:56",
  "path": "/api/me",
  "fieldErrors": [
    { "field": "email", "reason": "must not be blank" }
  ]
}
----

[NOTE]
====
`fieldErrors`는 주로 Validation 실패(400)에서만 채워지며, 그 외에는 빈 배열이거나 `null`일 수 있습니다.
====

==== COMMON

[cols="2,3,1",options="header"]
|===
|code |설명 |HTTP
|COMMON_BAD_REQUEST |잘못된 요청 |400
|COMMON_NOT_FOUND |리소스를 찾을 수 없음 |404
|COMMON_CONFLICT |충돌(중복 등) |409
|===

==== AUTH

[cols="2,3,1",options="header"]
|===
|code |설명 |HTTP
|AUTH_UNAUTHORIZED |인증 필요 |401
|AUTH_FORBIDDEN |권한 없음 |403
|AUTH_TOKEN_EXPIRED |토큰 만료 |401
|AUTH_REFRESH_REUSE |리프레시 토큰 재사용 |401
|===

==== USER

[cols="2,3,1",options="header"]
|===
|code |설명 |HTTP
|USER_DUPLICATE_EMAIL |이메일 중복 |409
|USER_DUPLICATE_HANDLE |핸들 중복 |409
|===

==== FILE

[cols="2,3,1",options="header"]
|===
|code |설명 |HTTP
|FILE_FORBIDDEN |다른 유저 파일 접근 |403
|===

==== FOLLOW

[cols="2,3,1",options="header"]
|===
|code |설명 |HTTP
|FOLLOW_SELF |자기 자신 팔로우 |400
|===
